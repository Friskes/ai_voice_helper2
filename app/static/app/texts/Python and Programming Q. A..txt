Что такое язык программирования Пайтон Python ?
Python это интерпретируемый высокоуровневый язык программирования с динамической строгой типизацией и автоматическим управлением памятью.






В чем отличие между интерпретируемыми и компилируемыми языками программирования ?
Главное отличие компилируемых языков от интерпретируемых в скорости выполнения программ в пользу компилируемых за счёт предварительной компиляции в машинный код.






Как происходит интерпретация в Пайтон Python ?
В python интерпретатор имеет компилирующий тип, то есть сначала компилятор переводит исходный код программы в байт-код, и затем интерпретатор выполняет этот байт-код.






В чём отличие файлов с расширением точка пай .py от файлов с расширением точка пайси .pyc ?
Когда файл с расширением .py запускается с использованием интерпретатора Python, он автоматически преобразуется в байт-код для выполнения и сохраняется в файл .pyc, чтобы его можно было повторно использовать из кеша если он не был изменён.






Что значит высокоуровневый язык программирования ?
Это язык с высоким уровнем абстракции, когда программист не задумывается о низкоуровневых процессах которые происходят под капотом языка.






Что такое динамическая типизация ?
Динамическая типизация означает то что тип (type) объекта может меняться по ходу выполнения программы.






Что такое строгая типизация ?
Строгая типизация означает то что мы не можем смешивать разные типы данных в одних выражениях.
Например складывать
int число со str числом.






Что значит автоматическое управление памятью ?
Автоматическое управление памятью означает то что если в коде программы пропадут все ссылки на объект, то сборщик мусора (Garbage Collector) очистит ячейку памяти с этим объектом.






Что такое Гил Джил GiL и какие проблемы он решает ?
Это Global Interpreter Lock, он блокирует интерпретатор так, что в один момент времени может исполнятся всего один реальный поток программы, также за счет GiL, интерпретатор может автоматически безопасно управлять памятью за счет наличия всего одного потока.






Что такое пеп восемь PEP8 ?
Это style guide того, как должен быть оформлен код, написанный на Python.






Как в Пайтон Python устроены переменные ?
Переменные в пайтон являются ссылками на объекты в памяти.






Какие существуют области видимости Пространства имен в Пайтон Python ?
built-in - переменные, функции и классы встроенные в интерпретатор, для их использования не нужно импортировать модули.

Оператор равенства сравнивает значения объектов, а оператор is сравнивает их адрес в памяти.








Какие типы данных могут быть ключами в словаре dictionary ?

Что возвращает метод словаря айтемс items ?
Список содержащий кортежи с парами ключ, значение.






Что такое хэш таблица hashtable ?
Это структура данных реализующая интерфейс ассоциативного массива. Позволяющая выполнять операции на поиск, вставку и удаление элемента за константное время. Для этого хеш-таблица использует массив с парами (ключ, значение) и функцию для хеширования ключей.
В Python хэш-таблицу называют словарем(dictionary).






Что такое ассоциативный массив ?
Это абстрактный тип данных, позволяющий хранить пары вида (ключ, значение) в ячейках называемых бакетами и поддерживающий операции поиска, вставки и удаления пар по ключу.






В чём отличие между словарем dict и списком list ?
Словарь содержит в себе пары ключ, значение, соответственно чтобы взять значение из словаря к нему нужно обращаться по ключу.
В списке каждое значение проиндексировано, индексация начинается с нуля, соответственно чтобы взять значение из списка к нему нужно обращаться по индексу.

Функция может принимать такие параметры как аргс Args неименованные, кваргс Kwargs именованные, что это за параметры и для чего они нужны ?

Что такое рекурсия и для чего она используется ?
Это механизм самовызова функции, используется например для изменения входных аргументов снова и снова, чтобы получить определенный результат, либо же для прохождения в глубину некоторой вложенной коллекции.






Что такое аннотации типов type hints и зачем они нужны ?
Это подсказки к аргументам, переменным, функциям и т.д.
Используются для того чтобы программист который читает код сразу понимал к какому типу принадлежит объект, а также современные среды разработки умеют показывать ошибки связанные с типизацией, и отображают всплывающие подсказки с информацией по использованию объектов.






Что такое лямбда lambda функции function и где они применяются ?
Это анонимные функции у которых нету имени, могут принимать неограниченное количество аргументов, всегда неявно возвращают результат своей работы.
Обычно применяются в функциях которые принимают аргументом функцию, например функции sorted, filter, map и т.д.






Для чего используется конструкция иф нейм равно мейн
if __name__ == "__main__": ?
Данная конструкция выполняет проверку, запущен ли скрипт напрямую или импортирован из другого модуля.
Если скрипт запущен напрямую, то значение переменной __name__ устанавливается в "__main__", что позволяет выполнить код внутри блока if.
Если скрипт импортирован из другого модуля, то значение __name__ устанавливается в имя этого модуля, и код внутри блока if не выполняется.






Что такое тернарный оператор и как он записывается ?
Это условие if else которое записывается в одну строку, сначало записывается  значение которое вернётся если условие верно, дальше ключевое слово if затем само условие, затем ключевое слово else и значение которое вернётся если условие не верно.






Что делает класс функция зип zip ?
Класс принимает на вход две или более последовательности и поэлементно группирует их в кортежи. Класс останавливается, когда заканчиваются элементы в одной из последовательности и возвращает генератор.






Что такое глубокое deep и поверхностное shallow копирование copy, зачем нужны, как их сделать ?
Поверхностное копирование это копирование только первого уровня  вложенности объекта, он будет размещен по новому адресу в памяти, все вложенные уровни будут ссылаться на исходный объект,
вызывается как обычный метод у объекта.
Глубокое копирование это копирование всех уровней вложенности объекта, он также будет размещен по новому адресу в памяти, все вложенные уровни будут ссылаться на новые объекты,
вызывается как функция из модуля copy в которую передается копируемый объект.






Что такое виртуальное окружение виртуаленв virtualenv венв venv, и зачем оно нужно ?
На компьютере есть глобальный пайтон который ставится в систему у него есть своя коллекция библиотек. Виртуальное окружение позволяет устанавливать необходимую версию пайтон и необходимые библиотеки в локальную область на компьютере которая видна лишь в директории виртуального окружения, тем самым появляется возможность ставить для каждого проекта свою версия пайтон и свою версию библиотек не засоряя при этом коллекцию системных библиотек.






Что такое итератор iterator ?
Это объект, который реализует магические методы __iter__, __next__ и может возвращать свои элементы по одному за раз. Запросить элемент можно используя функцию next или цикл for который под капотом выполняет это же действие.
Его можно создать с помощью функции iter() передав в нее последовательность.






Что такое итерируемый объект iterable object ?
Это любой объект способный возвращать итератор. Такими объектами например являются: списки, кортежи, строки, словари.






В чём отличие итератора iterator от генератора generator ?
Итератор — это просто механизм поэлементного обхода данных. Генератор — позволяет перебирать элементы коллекции, не загружая все элементы в память, что делает его очень эффективным для работы с большими наборами данных.






Что такое генераторы коллекций списков множеств словарей списковое включение или (лист list сет set дикт dict comprehensions) ?
Генераторы коллекций это более удобная и быстрая возможность генерировать коллекции чем конструкция с обычным циклом for.






Приведи пример использования генератора коллекций списков множеств словарей списковое включение или (лист list сет set дикт dict comprehensions) ?
Перебор в цикле генератора или итератора, либо последовательности для преобразования в необходимую коллекцию по условию(опционально).






Что такое выражение генератор ?
Это итерируемый объект который записывается как  генератор коллекций только в круглых скобках, может возвращать по одному элементу за раз при вызове функции next, либо при переборе в цикле for.






Что такое функция генератор ?
Это итерируемый объект который может возвращать по одному элементу за раз за счёт оператора йелд (yield) вместо ретурн (return).
При вызове функции next, либо при переборе в цикле for.






Что такое замыкание closure ?
Это механизм когда функция запоминает переменные из своей внешней области видимости, даже если эта область уже недоступна, реализуется с помощью двух вложенных функций.






Что такое декоратор decorator ?
Декоратор это паттерн проектирования он основывается на том что мы создаём функцию декоратор с помощью которой мы можем декорировать (модифицировать) другую функцию, тем самым расширять ее функционал без прямого вмешательства в код написанный в декорируемой функции.






Что такое контекстный менеджер context manager ?
Это конструкция with . . as . .
которая позволяет обращаться к объекту и получить из него другой объект, и работать с ним внутри открытого контекста.








В чем главная польза контекстного менеджера context manager ?
В автоматическом закрытии файла в случае возникновения исключения.






Как создать свой контекстный менеджер context manager?
Необходимо определить класс, который содержит магические методы __enter__ и __exit__
Метод __enter__ выполняется перед выполнением блока кода внутри оператора with и возвращает экземпляр класса контекстного менеджера либо копию объекта переданного при создании экземпляра менеджера.
Метод __exit__ выполняется после завершения выполнения блока кода with.






Чем отличается модуль от пакета ?
Модуль это один файл с расширением .ру
Пакет это директория которая содержит пайтон модули с расширением .py, а так же файл __init__.py, в современных версиях пайтон последнее условие не является обязательным.






Для чего нужен, когда выполняется, и как много раз будет выполнен файл инит точка пай __init__.py ?
Этот файл необходим для того чтобы указать в нем код который должен быть выполнен при импорте пакета или модуля из этого пакета. Выполняется один раз когда пайтон натыкается на его импорт.






В чем различие цикла фор for и цикла вайл while ?
Цикл for используется, когда заранее известно количество итераций.
Цикл while используется, когда количество итераций заранее неизвестно, к тому же итерации будут продолжаться пока условие истинно.

Как оставить в списке list только элементы с уникальными значениями и при этом оставить изначальный порядок элементов?
Использовать приведение к словарю т.к. словарь в пайтон сохраняет порядок элементов, затем получить ключи словаря и преобразовать получившуюся последовательность обратно к списку.










Что такое слотс __slots__ и для чего он нужен?
Это коллекция которая позволяет ограничивать список атрибутов экземпляра класса в котором определена. Не имеет влияния на атрибуты самого класса.






Что такое дескриптор?
Это класс, который реализует магические методы __set_name__, __set__, __get__ и __delete__ для атрибутов другого класса, которые позволяют менять поведение по умолчанию при работе с этими атрибутами.






Что такое асинхронность async и для чего она нужна?
Асинхронность это способность программы переключаться между задачами в рамках одного потока.
Например если задача ожидает ответа при HTTP запросе или от Базы Данных, программа может временно переключится на выполнение другой задачи до тех пор пока не придёт ответ от первой задачи.






Что такое корутины coroutine ?
Это объекты которые работают асинхронно, то есть по очереди. Корутина может становиться на паузу, чтобы дать другим корутинам немного поработать, в этом заключается принцип асинхронности.






Что такое асинк async и эвэйт await?
Асинк используется для объявления асинхронной функции, это означает что функция будет возвращать объект coroutine(корутину) которую можно выполнить асинхронно.
Эвэйт используется внутри асинхронной функции для ожидания результата выполнения другой асинхронной операции(корутины).






Что такое многопоточность?
Многопоточность это способность разделять задачи на несколько одновременно выполняющихся потоков. В Python она реализована быстрым переключением виртуальных потоков в рамках одного реального.
За счет многопоточности программа может оставаться отзывчивой, например пока один поток ожидает ввода пользователя, а другой одновременно выполняет задачу по отрисовке графического интерфейса.






Что такое мультипроцессинг?
Модуль multiprocessing позволяет распределять вычисления по нескольким ядрам процессора.
Это называется параллелизмом.






Что такое исключения exceptions ?
Это механизм который позволяет управлять приложением при возникновении ошибок а не аварийно завершать его работу.






Какие существуют виды исключений exceptions ?
В Python исключения представлены в виде иерархии классов, на самом верху находится BaseException, разновидностей исключений очень много, но вот несколько основных:
SyntaxError
TypeError
ValueError
NameError
KeyError
ImportError






Как записывается обработка исключений exceptions ?
С помощью конструкции
try .. except .. esle .. finally ..
try - выполняется всегда, в нем мы и отлавливаем исключение.
except - выполняется если в блоке try произошло исключение.
else - выполняется в том случае если try выполнится без исключения.
finally - выполняется в любом из вариантов.






Зачем нужны исключения exceptions ?
Использование исключений позволяет разработчикам создавать надежные программы, которые продолжают работать даже при возникновении непредвиденных ошибок.






Зачем нужно писать юнит unit модульные тесты tests ?
Тесты, проверяют отдельные блоки и функции написанного кода. Нужны, чтобы быстро протестировать написанный фрагмент кода и сразу сразу обнаружить проблему если таковая имеется.






Какие виды тестов существуют ?
Юнит-тест(он же модульный) это тест который тестирует небольшой фрагмент кода, например класс, или функцию.
Интеграционный тест это тест который тестирует работу проекта в его рабочем окружении, проверяет что все компоненты приложения корректно работают и обмениваются между собой информацией.






Какие существуют инструменты в Пайтон Python для тестирования?
В сам пайтон уже встроен модуль unittest(юнит тест), еще существует модуль pytest(пай тест) который необходимо устанавливать отдельно.






Что такое тидиди тдд TDD ?
Это разработка программ на основе тестов.
То есть сначала пишутся тесты и уже на их основе пишется бизнес логика приложения.






Что такое ООПЭ OOP и зачем оно нужно?
Это парадигма разработки, набор правил и критериев, по которым пишут код. Её суть в том, что весь код состоит из объектов, которые взаимодействуют друг с другом.






В ООПЭ OOP выделяют четыре основных базовых принципа, назови их?
Абстракция, Наследование, Полиморфизм, Инкапсуляция.






Что такое абстракция ?
Это способ снизить сложность программного обеспечения за счет сокрытия общих характеристик и функциональности за более простым API.






Что такое наследование ?
Наследование позволяет создать дочерний класс на основе родительского класса, при этом все публичные атрибуты и методы из родительского класса становятся доступны в дочернем классе.






Что такое композиция ?
Композиция позволяет переиспользовать код без необходимости наследования






Когда лучше использовать наследование а когда композицию ?
Наследование лучше использовать
когда необходим весь функционал родительского класса.
Композицию лучше использовать когда необходимо получить лишь часть реализации другого класса.






Что такое полиморфизм ?
Полиморфизм это переопределение методов родительского класса в дочернем классе для модификации их работы, получается что методы с одним именем но в разных классах будут иметь разное поведение, это и называется полиморфизмом.






Что такое инкапсуляция ?
Инкапсуляция позволяет сделать объект _защищённым либо __приватным, к __приватному объекту нельзя получить доступ вне класса если конечно не прибегать к обходному способу:
экземпляр._класс__объект
Либо:
класс._класс__объект






Что такое селф self в параметрах методов method ?
self это ссылка на экземпляр текущего класса.






Что такое супер super и для чего он нужен ?
Это ссылка на первый родительский класс в списке иерархии классов.
Он нужен для доступа к родительским методам, из переопределенных методов в дочернем классе.
Используется внутри  переопределенных методов для их перегрузки, то есть для добавления нового функционала без необходимости полного копирования(копипаста) содержимого родительского метода.






Что такое метод инит __init__ в классах ?
Инит это магический метод конструктор, который служит для инициализации аргументов переданных в класс.






Где создаётся объект object экземпляра класса ?
Объект экземпляра класса создаётся в магическом методе нью __new__






Что такое статик метод static method ?
Статик метод по сути является обычной функцией которая не принимает объект экземпляра класса (self), но находится внутри класса т.к. тематически связан с классом.








Что такое класс метод class method ?
Класс метод принимает первым аргументом класс (cls) а не экземпляр класса (self), и служит для работы с объектами внутри своего класса.






Что такое декоратор проперти decorator property ?
Проперти используется для декорирования метода который становится атрибутом класса, значение которого можно получить без использования круглых скобок в отличие от обычного метода.






Что такое геттеры, сеттеры и делитеры getter setter deleter ?
Это специальные надстройки над декоратором проперти которые позволяют получать, устанавливать и удалять данные.


Геттером по умолчанию считается метод обернутый в декоратор проперти, для создания сеттера и делитера необходимо создать 2 отдельных метода с именем первого метода и обернуть их в декоратор сеттер и делитер.






Какая разница между методами дандер стр __str__ и дандер репр __repr__ и какой из них использовать ?
Оба метода отвечают за строковое представление объекта,
__str__ отвечает за пользовательское представление объекта,
__repr__ отвечает за программное представление объекта, то есть для программиста.






Что такое абстрактный класс abstract class и для чего он используется ?
Это концепция объектно ориентированного программирования. А также хорошая практика принципа "не повторяйся". Полезно использовать абстрактный класс для определения общих методов и атрибутов которые должны быть реализованы в наследниках.
У абстрактного класса нельзя создать объект экземпляра класса.






Что такое мета класс meta class и для чего он используется ?
Мета класс является шаблоном для классов, на основе мета класса строятся другие классы, мета класс используется для того чтобы перехватить создание класса и изменить его, либо изменить поведение класса при создании его экземпляров.
У мета класса есть 4 основных метода:
__prepare__ подготавливает данные затем они попадают в new.
__new__ затем в init,
__init__ затем в call,
__call__ отвечает за поведение при вызове класса, возвращает экземпляр класса.






Что такое эмэро MRO Method Resolution Order, Порядок Разрешения Методов ?
MRO позволяет выяснить, из какого родительского класса будет вызван метод, тем самым решается так называемая Diamond(Даймонд) problem.


Это список который лежит внутри класса содержащий все родительские классы в их иерархическом порядке, его можно посмотреть вызвав метод .mro()






Как называется объект от которого наследуются абсолютно все объекты в Пайтон Python ?
PyObject






Какие два поля fields хранит в себе пай обжект PyObject ?
Тип (type) объекта
Счётчик ссылок на ячейку в памяти






Зачем нужен счетчик ссылок ?
Счётчик ссылок нужен для того чтобы сборщик мусора GiL понимал, удалять данный объект из памяти или он всё ещё используется в программе.






Что такое мидлвейр Middleware промежуточное программное обеспечение в джанго django ?
Middleware позволяет обрабатывать запросы из браузера, прежде чем они достигнут представления(View), а также ответы от представлений до того, как они возвратились в браузер.






Что происходит после ввода адреса в строке браузера ?
Браузер смотрит сохранен ли ip запрашиваемого адреса в кэше, если нет, тогда браузер обращается к DNS серверу для получения айпи адреса сервера который привязан к доменному имени который мы ввели в строке браузера, когда айпи адрес получен, браузер устанавливает TCP/IP соединение с сервером и отправляет HTTP GET запрос к серверу чтобы получить контент страницы сайта, сервер обрабатывает запрос и возвращает контент запрашиваемой страницы и браузер отрисовывает полученные данные.






Что такое тисипи айпи TCP/IP ?
Это набор сетевых протоколов которые задают стандарты связи между компьютерами.






Что такое тисипи TCP ?
TCP - Протокол транспортного уровня, с установкой соединения. С гарантированной доставкой пакетов, если пакет не дошел до получателя за определенный таймаут, то он посылается повторно.






Что такое юдипи UDP ?
UDP - Протокол транспортного уровня, без установки соединения. С негарантированной доставкой пакетов.






Что такое эйчтитипи HTTP ?
Это протокол описывающий взаимодействие между клиентом и сервером на основе запросов.






Какие эйчтитипи HTTP запросы существуют ?
HEAD - получить заголовки, идентичные тем что возвращаются при GET запросе.
GET - получить данные от сервера
POST - передать данные на сервер
PUT - изменить данные полностью
PATCH - изменить данные частично
DELETE - удалить данные
OPTIONS - получить разрешённые методы запросов от сервера.






Какие бывают статус коды ?
100-е - информационные (info)


200-е - успешные (success)
200 - Запрос успешно обработан.
201 - Успешное создание данных.
204 - Нет содержимого для возврата клиенту.


300-е - перенаправления (redirect)
301 - Адрес перемещен навсегда.
302 - Адрес перемещен временно.


400-е - клиентские ошибки
400 - Сервер не понял запрос.
401 - Требуется авторизация.
403 - Доступ запрещен.
404 - Адрес не найден.


500-е - серверные ошибки
500 - Внутренняя ошибка сервера.
501 - Метод запроса не поддерживается сервером.






Что такое идемпотентность ?
Это операция, которая при многократном вызове возвращает один и тот же результат.
Она применима к методам запроса: GET, HEAD, PUT и DELETE.






Что такое эсесель SSL Secure Sockets Layer ?
Это криптографический протокол, который подразумевает безопасную связь. Он позволяет устанавливать зашифрованное соединение между веб-сервером и веб-браузером.






Что такое вебсокет websocket ?
Это протокол связи предназначенный для обмена сообщениями между клиентом и веб сервером в режиме реального времени. В отличие от HTTP, вебсокеты позволяют работать с двунаправленным потоком данных, то есть как клиент может отправить сообщение серверу, так и сервер клиенту.






Что такое Куки Cookies ?
Это небольшой фрагмент данных, хранимый на компьютере пользователя. Всякий раз при попытке открыть страницу соответствующего сайта, браузер пересылает этот фрагмент данных веб-серверу в составе HTTP-запроса






Для чего нужны Куки Cookies?
Cookies нужны для сохранения данных о предыдущих посещениях сайта, в них может храниться айди сессии, csrftoken, настройки локализации и т.д.






Что такое корс CORS политика ?
Это механизм, использующий HTTP-заголовки, для того чтобы дать возможность JavaScript приложению в браузере получать разрешения на доступ к ресурсам с сервера на домене, отличном от того, на котором расположен сайт.

Что такое оэрэм ORM ?
Это объектно реляционное отображение которое связывает пайтон объекты с таблицами в базе данных.






С какими оэрэм ORM ты работал ?
Django ORM,
SQLAlchemy ORM.














Какие существуют виды связей между таблицами в реляционной базе данных ?
One to one - один к одному


One to many - один ко многим


Many to many - многие ко многим, реализуется с помощью связующей третьей таблицы которая содержит ключ первой и второй таблицы.
В Django ORM третья таблица генерируется автоматически.






Для чего нужны и в чём отличие префетч релейтед prefetch_related от селект релейтед select_related ?
Данные методы Django ORM позволяют избавиться от побочных запросов в базу данных.
Например могут помочь решить известную проблему N + 1 из за которой для каждой связанной сущности генерируется дополнительный запрос в базу данных.


select_related полезен, когда работаешь с отношениями «один к одному» и «один ко многим», он создает JOIN.


prefetch_related полезен при работе с отношениями «один ко многим» и «многие ко многим», он создает дополнительный запрос для каждого поля переданного в параметре, а объединение данных выборок происходит на уровне Python.






Что такое праймари кей первичный ключ PrimaryKey ?
Это ключ который идентифицирует уникальную строку в таблице.






Что такое форейгн кей ForeignKey ?
Это класс в Django ORM, который используется для создания One to Many связи между таблицами.






Для чего используется параметр релейтед нейм related_name в классе форейг кей ForeignKey ?
Для указания имени по которому модель в которой объявлен ForeignKey будет доступна из связанной модели.






Что такое индексы в таблицах ?
За счёт индексов база данных знает где точно лежит строка, используется для ускоренного доступа к строке в проиндексированном столбце таблицы.


Однако индексирование также может обладать и недостатками, например, увеличение требований к объему диска и замедление выполнения операций вставки и изменения за счет того что дополнительно необходимо перестраивать индексы.


По умолчанию индексы хранятся в формате B-tree(сбалансированного дерева).
В Django ORM можно создавать индекс как для одного столбца так и для сочетания двух и более столбцов)






Какие существуют категории джойнов join в базе данных ?


LEFT - Возвращает все строки из левой таблицы и строки совпадающие по условию ON из правой таблицы.


RIGHT - Возвращает все строки из правой таблицы и строки совпадающие по условию ON из левой таблицы.


INNER - Возвращает строки, имеющие совпадения по условию ON из обеих таблиц, то есть имеющие общее значение в обеих таблицах.


FULL - (выполняется по умолчанию, если не указать конкретный JOIN) - Возвращает все строки, из обеих таблиц даже при отсутствии совпадений.


CROSS - Возвращает объединение каждой строки левой таблицы с каждой строкой правой таблицы.


OUTER - Возвращает строки, имеющие совпадения по условию ON из обеих таблиц, то есть имеющие общее значение в обеих таблицах, а также строки из внешней таблицы неподходящие по условию, со значением NULL.






Что такое транзакция transaction ?
Это набор операций по работе с базой данных объединенных в один неделимый блок.






Какой смысл паковать запросы в транзакцию transaction ?
Она защищает данные от ошибок благодаря принципу «всё, или ничего».
Или будут выполнены все запросы, которые были упакованы в транзакцию, или ни один.






Что такое асид эйсид ACID ?
Это набор требований к транзакционной системе, которые обеспечивают сохранность данных.


A - (Atomicity, Атомарность) - гарантирует что транзакция будет выполнена полностью или не будет выполнена совсем.
C - (Consistency, Согласованность) - благодаря тому, что транзакция не допускает промежуточных результатов, база остается Согласованной.
I - (Isolation, Изолированность) - во время выполнения транзакции, параллельные транзакции не должны оказывать влияния на ее результат.
D - (Durability, Надёжность) - Если транзакция выполнена, никакие сбои в работе системы не повлияют на эту транзакцию.






Что такое изоляция транзакций ?
Изолированность транзакций отражает то, насколько сильно влияют друг на друга параллельно выполняющиеся транзакции.






Что такое эйпиай апиай апи API ?
API это разновидность интерфейса предназначенная для взаимодействия программ между собой.






Что такое рест REST и рестфул эйпиай апиай апи RESTFUL-API ?
REST это набор правил того, как программисту организовывать написание кода серверного приложения, чтобы все системы легко обменивались данными и приложение можно было легко масштабировать.


RESTFUL это когда все правила соблюдены.


1. Разграничение клиента и сервера.
Отделение потребностей клиента от сервера хранящего данные.


2. Отсутствие записи состояния клиента.
Сервер не должен хранить информацию о состоянии клиента, запросы клиента должны содержать только информацию необходимую для получения данных.


3. Кэшируемость.
Использование кэширования способно устранить некоторые клиент-серверные взаимодействия.


4. Единообразие интерфейса.
Единый интерфейс позволяет каждому из сервисов развиваться независимо, все данные должны запрашиваться через один URL-адрес протоколом HTTP.


5. Слои.
Применение промежуточных серверов способно повысить масштабируемость за счет балансировки нагрузки и распределенного кэширования.






Что такое соа сервис ориентированная архитектура SOA Service Oriented Architecture ?
Сервис ориентированная архитектура - это метод разработки программного обеспечения, который использует программные компоненты, называемые сервисами, для создания приложений.






Что такое спа сингл пейдж апликейшен SPA Single Page Application ?
это веб-приложения, которые  для изменения контента страницы, вместо перезагрузки страницы, используют технологию AJAX для отправки запроса и получения новых данных от сервера.






Чем отличается микросервисная архитектура от монолитной ?
В монолитной архитектуре все компоненты программы собраны в рамках одного приложения, то есть работа с базой данных, работа с представлением, логикой и т.д.
В микросервисной архитектуре вся логика будет поделена между различными микро приложениями (сервисами) которые могут находятся на разных серверах.






Что такое эм ви си MVC Model View Controller модель представление контроллер ?
Это паттерн Model-View-Controller для разделения данных приложения и управляющей логики на три отдельных компонента: модель, представление и контроллер — таким образом, что модификация каждого компонента может осуществляться независимо.






Что такое эм ви пи MVP Model View Presenter модель представление контроллер ?
Почти тоже самое что и MVC, Controller переименовывается в Presenter.






Что такое эм ви ти MVT Model View Template модель представление тимплейт ?
Работа Django основана на паттерне M.V.T. «Модель, Представление, Шаблон». Смысл этого паттерна в том, чтобы разделить логику работы сайта, внешний вид страниц и реакцию сервера на запросы пользователей.






Что такое принцип драй DRY Dont repeat yourself ?
Don't repeat yourself(Не повторяйся)
Это принцип разработки программного обеспечения, нацеленный на снижение дублирования кода.






Что такое принцип кисс KISS Keep it simple ?
Keep it simple, stupid(Делай проще, тупица)
Надо писать программный код максимально простым и понятным, чтобы он был понятен как себе так и другим разработчикам.
Поэтому при проектировании программ следует избегать излишней сложности.






Что такое принципы солид SOLID ?
S (Single Responsibility) - Принцип единой ответственности.
(Для каждого класса должно быть определено единственное назначение)
Один класс должен решать всего одну задачу, не должно быть классов которые содержат в себе одновременно к примеру работу с сетью, работу с файлами, работу с визуальным выводом и т.д.
Суть этого принципа в декомпозиции.
Класс который содержит в себе множество обособленных методов называют объектом бога.


O (Open Closed) - Принцип открытости / закрытости.
Классы должны быть открыты для расширения но закрыты для изменения.
То есть лучше расширить функционал класса унаследовав его чем делать изменения прямо в нём.


L (Liskov Substitution) - Принцип подстановки Барбары Лисков.
Наследуемый класс должен дополнять а не замещать поведение родительского класса, то есть методы переопределенные в дочернем классе должны расширять функционал а не замещать его.


I (Interface Segregation) - Принцип разделения интерфейса.
(Много интерфейсов специального назначения лучше чем один интерфейс общего назначения)
Лучше разделять классы на узкоспециализированные, решающие только одну задачу.
Дочерние классы не должны содержать методы родительских классов, которые они не должны использовать.


D (Dependency Inversion) - Принцип инверсии зависимостей.
Классы должны зависеть от абстракций, а не от конкретных деталей.

Что такое жвт JWT джейсон веб токен JSON Web Token и как он работает ?
Это строка в формате: header.payload.signature которая служит для прохождения авторизации.
При аутентификации пользователя, сервер создаёт и возвращает клиенту access_token(с малым временем жизни) и refresh_token(с большим временем жизни).


При повторных запросах пользователь должен добавлять в заголовок запроса полученный access_token для прохождения авторизации и получения доступа к данным.


Когда время жизни access_token истечет, тогда при очередном запросе к серверу будет необходимо отправить access_token вместе с refresh_token, если оба токена корректны, клиент пройдет авторизацию и сервер вернёт новые access_token и refresh_token.






В чем отличие Аутентификации и Авторизации ?
Аутентификация - это процесс проверки учетных данных.
Авторизация - это процесс проверки прав на осуществление действий.
Например аутентификация происходит при логине, а авторизация при всех последующих запросах.

Что такое аджайл Agile канбан Kanban скрам скрум Scrum сейф SAFe ? 
Это система разработки, в которой сложные задачи разбиваются на более простые. Команда постепенно выдает готовые части продукта, их тестируют и оценивают. Одну простую задачу называют спринтом. В конце спринта команда подводит итоги и ставит себе следующие задачи.


Главные принципы Agile:
1) Люди и их взаимодействие важнее процессов и инструментов.
2) Работающий продукт важнее исчерпывающей документации.
3) Готовность к изменениям важнее следования первоначальному плану.


Есть два подхода к работе над проектом, основанные на Agile:


Методика Kanban (команда 3-9 человек) — это способ визуализации задач с помощью досок, на которых задачи располагаются в соответствии со статусом.
Стандартная канбан-доска делится на три колонки:
to do — список задач,
in progress — задачи, которые взяли в работу и выполняют,
done — завершенные задачи.


Методика Scrum (команда 3-9 человек) — это способ визуализации задач с помощью досок, на которых задачи располагаются в соответствии со статусом.
Отличие от Kanban в том, что самую важную роль играют спринты, и задача не может находиться в работе дольше, чем длится спринт.

Что такое клик хаус ClickHouse ?
Это система управления базами данных (СУБД) с открытым исходным кодом, используемая для аналитической обработки больших данных в режиме онлайн.


Ключевым преимуществом Кликхаус считается высокая скорость выполнения SQL-запросов на чтение, которая обеспечивается благодаря следующим архитектурным особенностям:


столбцовое хранение данных, что позволяет считывать данные только из нужных колонок и эффективно сжимать однотипную информацию;
физическая сортировка данных по первичному ключу позволяет быстро получать конкретные значения или диапазоны;
векторные вычисления по кусочкам столбцов снижают издержки на диспетчеризацию и позволяют более эффективно использовать CPU;
распараллеливание операций как в пределах одного сервера на несколько процессорных ядер, так и в рамках распределенных вычислений на кластере за счет механизма шардирования;
поддержка приближенных вычислений на части выборки, что снижает число обращений к жесткому диску и еще больше повышает скорость обработки данных.


Благодаря высокой скорости генерации аналитических отчетов по большим данным в режиме реального времени, ClickHouse наиболее востребован в следующих областях:


веб-аналитика и контекстная реклама;
real time мониторинг бизнес-метрик, например, анализ потребительского поведения на сайте;
интерактивное взаимодействие с пользователями, к примеру, онлайн-игры;
контроль технических показателей, между физическими объектами;
реализация корпоративных хранилищ данных.

Для добавления данных в БД необходимо сначала создать объект модели, который передается в метод add() объекта Session. После добавления для подтверждения изменений у объекта Session вызывается метод commit().






Что такое алембик Alembic ?
Это инструмент для миграции базы данных, используемый в ORM SQLAlchemy.
Миграция базы данных — это своего рода система контроля версий для баз данных.


При разработке приложения распространена практика изменения схемы таблицы в БД, из-за изменения старых полей моделей, или добавления новых моделей. Этим можно легко управлять с помощью Alembic.


Он, позволяет менять схему базы данных при развитии приложения. Он также следит за изменениями самой базы, так что можно как накатывать новые изменения так и откатываться к старым.
Если не использовать Alembic, то за всеми изменениями придется следить вручную.






Что такое пайдантик пидантик Pydantic ?
Это библиотека, построенная на основе классов данных Python, добавляющая дополнительные функции для валидации и анализа данных на основе типизации.






Что такое проблема эн плюс один N + 1 ?
Эта проблема возникает, когда делаешь запрос к БД и вместо использования одного запроса с оператором JOIN дополнительно для каждого запроса генерируется побочный подзапрос в БД.


Например, если у нас есть модель подписки, в которой определена связь один ко многим с моделью пользователя.
Мы сделаем запрос для получения подписки и при попытке обратится к атрибуту пользователя у объекта подписки, будет сгенерирован дополнительный запрос в БД который вытащит объект пользователя.






Что такое агрегация и агрегирующие функции в базе данных БД ?
Агрегация позволяет вынести некоторые расчеты из Пайтон на уровень базы данных, то есть можно произвести некоторые математические расчеты прямо в базе данных, например просуммировать результат выборки или посчитать среднее значение у списка чисел.






Что такое граф ку эль GraphQL ?
Это язык запросов а также серверная среда для API созданная для простого и эффективного обмена данными между клиентом и сервером. В отличие от REST, который использует множество эндпоинтов для доступа к данным, GraphQL использует единый эндпоинт.






Что такое графана Grafana ?
Она предназначена для отображения всевозможных циклических метрик. Помимо предопределенных системных метрик о ЦП, памяти и т.д, можно сконфигурировать любой набор произвольных метрик.
